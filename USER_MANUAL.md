# Python 文件智能整理程序 - 用户手册

## 1. 简介
本程序是一款功能强大的文件整理工具，旨在通过自动化方式解决文件杂乱问题。它支持基于文件类型和关键字的智能分类，同时提供安全备份、撤销操作和自定义规则配置。
**新特性**：采用非递归扫描模式，保护子目录结构；新增快捷方式和应用程序的专属分类通道。

## 2. 系统要求
- **操作系统**: Windows 10/11, macOS, Linux
- **Python版本**: Python 3.6 或更高版本
- **依赖库**: 仅使用 Python 标准库，无需安装第三方包。

## 3. 安装步骤
1.  下载或克隆本项目代码到本地目录。
2.  确保您的系统已安装 Python (可在终端运行 `python --version` 检查)。
3.  解压项目文件（如果是压缩包）。

## 4. 快速开始

### 4.1 命令行模式
打开终端（CMD 或 PowerShell），进入项目根目录，运行以下命令：

**整理特定文件夹：**
```bash
python main.py "C:\Users\YourName\Downloads"
```
*注意：请将路径替换为您实际需要整理的文件夹路径。*

**启动交互式菜单：**
```bash
python main.py -i
```

**撤销上一次操作：**
```bash
python main.py -u
```

### 4.2 交互式菜单功能
运行 `python main.py -i` 后，您将看到以下选项：

1.  **Organize a Folder**: 输入目标文件夹路径，程序将自动扫描并整理。
2.  **Add Keyword Rule**: 添加关键字规则。
    - *示例*：输入关键字 `report`，对应文件夹 `WorkReports`。所有包含 "report" 的文件名都会被移动到 "WorkReports" 文件夹中。
3.  **Add File Type Rule**: 添加文件扩展名规则。
    - *示例*：输入扩展名 `psd`，对应类别 `Design`。所有 .psd 文件将被归类到 "Design" 文件夹。
4.  **Undo Last Operation**: 撤销最近一次整理操作，将文件恢复原位。
5.  **Exit**: 退出程序。

## 5. 配置指南

程序会在首次运行时生成 `config.json` 文件。您可以直接编辑此文件来调整高级设置。
项目根目录提供了一个 `config.sample.json` 作为参考模板。

### 核心规则说明

1.  **非递归扫描**：程序仅整理目标文件夹下的第一层文件，**不会**进入任何子文件夹。
2.  **分类优先级**：
    *   **第一优先级 - 特殊文件**：快捷方式 (.lnk) 和应用程序 (.exe) 会被强制移动到 `Shortcuts` 和 `Applications` 文件夹，即使它们的文件名包含关键字。
    *   **第二优先级 - 关键字**：如果文件名包含配置的关键字（如 "report"），则归类到对应文件夹。
    *   **第三优先级 - 扩展名**：根据文件后缀名（如 .jpg -> Images）进行归类。

### 配置文件结构说明
```json
{
    "special_types": {
        "Shortcuts": ["lnk", "url"],   // 最高优先级：快捷方式
        "Applications": ["exe", "msi"] // 最高优先级：应用程序
    },
    "file_types": {
        "Documents": ["pdf", "docx"],  // 定义类别和对应的扩展名列表
        "Images": ["jpg", "png"]
    },
    "keywords": {
        "invoice": "Financial"         // 定义关键字及其目标文件夹
    },
    "exclude_dirs": [".git", "venv"],  // 扫描时忽略的文件夹名称
    "exclude_extensions": ["tmp"],     // 不进行整理的文件后缀
    "backup_enabled": true,            // 是否启用备份（预留功能）
    "log_file": "organizer.log",       // 日志文件路径
    "undo_file": "undo_log.json"       // 撤销记录文件路径
}
```

## 6. 常见问题与故障排除 (Troubleshooting)

### Q1: 运行程序时提示 "PermissionError"？
**原因**: 程序试图移动正在被其他程序打开的文件，或者没有目标文件夹的写入权限。
**解决方法**: 
- 关闭所有可能正在使用目标文件的应用程序。
- 尝试以管理员身份运行终端。

### Q2: 整理后找不到文件了？
**原因**: 文件被移动到了相应的分类文件夹中。
**解决方法**: 
- 检查目标目录下的分类子文件夹（如 Documents, Images）。
- 查看 `organizer.log` 日志文件，其中记录了详细的文件移动路径。
- 使用撤销功能 `python main.py -u` 恢复文件。

### Q3: 为什么有些文件没有被整理？
**原因**: 
- 文件扩展名未在配置中定义，且文件名不包含任何关键字。
- 文件位于被排除的目录中（如 `.git`）。
- 文件扩展名在 `exclude_extensions` 列表中。
**解决方法**: 
- 检查 `config.json` 设置。
- 这些文件会被归类到 `Others` 文件夹（除非被显式排除）。

### Q4: 如何恢复默认设置？
**解决方法**: 删除根目录下的 `config.json` 文件，重新运行程序即可自动生成默认配置。

### Q5: 如果出现重名文件怎么办？
**机制**: 程序具有安全重名处理机制。如果目标文件夹已存在同名文件 `file.txt`，新文件将被重命名为 `file_1.txt`，确保不会覆盖原有文件。

## 7. 高级功能
- **多线程处理**: 程序使用多线程技术，在处理数千个文件时也能保持流畅。
- **日志记录**: 所有操作都会记录在 `organizer.log` 中，方便审计和追踪。
